# Security Policy

## Supported Versions

This project is early-stage. Security fixes target the latest version on the default branch.

## Reporting a Vulnerability

Please do not open a public issue for sensitive security reports.

Until a dedicated security contact exists, report privately to the maintainer listed in `MAINTAINERS.md`. Include:

- affected file or package
- impact
- reproduction steps
- suggested fix, if known

## Security Scope

This project is mostly documentation, templates, schemas, and local TypeScript helpers.

Security-sensitive issues may include:

- examples that encourage private data misuse
- templates that normalize spam or deception
- package scripts that run unexpected commands
- dependency vulnerabilities
- generated artifacts that leak secrets

## Out of Scope

- requests to add spam automation
- requests to bypass platform rules
- requests to scrape private data
- claims about guaranteed revenue or financial outcomes

## Maintainer Response

Maintainers should acknowledge credible reports, assess impact, prepare a fix, and document the resolution in `CHANGELOG.md`.
