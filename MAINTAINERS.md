## Maintainer Responsibilities

Maintainers are responsible for:

- protecting the ethical scope of the project
- reviewing issues and pull requests
- rejecting spam, scam, or income-guarantee contributions
- keeping tests and build checks passing
- updating `CHANGELOG.md` for notable changes
- reviewing security reports

## Review Standard

Before merging a contribution, maintainers should ask:

- Does it help people design or deliver useful offers?
- Does it avoid guaranteed income claims?
- Does it avoid spam automation?
- Does it avoid private data misuse?
- Is the scope small enough for this toolkit?
- Are docs and tests updated when needed?

## Release Notes

For each release:

1. Update `CHANGELOG.md`.
2. Run `npm test`, `npm run build`, and `npm audit`.
3. Tag the release.
4. Publish release notes with ethical scope and known limitations.
